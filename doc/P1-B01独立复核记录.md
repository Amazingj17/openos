## 7. 限制与结论

- 当前结果只证明固定数据字节、显式 projection、划分和校验链路可复现，不证明完整 GrapheonRL capability constraints 已复现。
- 主 pipeline 仍使用合成 smoke；P1-A01 必须只用 train 生成 HEFT teacher、只用 validation 选模，并阻止 test 进入训练/早停/超参数接口。
- input record 的许可依据是随 record 固定的许可证文件；上游 GitHub 代码和未声明 rights 的结果 record 仍不得导入。
- 公开 test 名单可审计但不可保密；最终比赛结果还需要隐藏/保留集或额外 OOD 证据。

复核结论：P1-B01/P1-01 的公开数据接入、许可边界、projection v1 和冻结划分满足验收条件；P0-07 的去重、checksum 与防泄漏检查同时收口。三项状态改为“已完成（A 复核通过）”，下一主任务为成员 A 的 P1-A01：HEFT teacher、BC 稳定基线和学习诊断。
