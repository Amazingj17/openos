## 4. 与冻结 MLP 的配对统计

- 同 seed、同 Scenario hash 的逐实例配对；
- `delta = task_gnn_ratio - masked_mlp_ratio`，越低越好；
- tie tolerance `1e-9`；
- seeds 和共享 Scenario ID 双层有放回 bootstrap；
- 10,000 次采样，bootstrap seed `20260717`；
- validation-only，test 不加载。

## 6. 决策与边界

P1-A03 证明一层 task-GNN 在固定 validation 上具有有利点估计，合法性、可恢复性和工程成本均可接受；但分层配对 CI 跨 0，未达到“固定 validation 上有稳健配对优势”的选择标准。

因此本轮决策为：

1. 主模型继续保留已独立复核的 14 维 masked MLP；
2. task-GNN 标记为“方向性改善、未证实”，不作为主结果候选；
3. 不追加 resource-GNN、课程、新奖励或更多 validation 调参来补救；
4. 公开 test、OOD 和竞赛隐藏测试继续不访问；
5. 用户推送后由成员 B 从新不可变提交复核正式 artifact、配对脚本、统计和停止决策，通过后关闭 P1-A03。

这个结论不是“GNN 无效”，而是当前 3-seed/30-scenario validation 证据不足以支持替换 MLP。任何最终领先表述仍须等待独立 test/OOD 门禁。
