## 1. 不可变边界

本规程只执行已预注册的候选，不再改变 seed、场景、模型、特征、奖励、优化器、epoch、transition 预算或 checkpoint selection。必须同时满足：

1. `origin/main` 已包含 `configs/p1_a05_implementation_review.json`；
2. 正式 HEAD 与远端一致，tracked 工作树干净；
3. `trisched/`、生产配置和依赖相对批准实现 `2ab2c66...` 零差异；
4. `outputs/p1-a05-size-robustness/` 尚不存在；
5. prepared manifest 与 dry-run 仍为 receipt 绑定的 `3fe388b3...1a96fa` 和 `b81fde38...87b2a`；
6. 公开 STG test 原始字节、archive 和 gate receipt 均不存在于训练/评测根。

不满足任一项就停止。不得运行 `claim-test-gate`，不得把开发切片用于梯度或选模。

## 2. 推送后正式预检

```powershell
git fetch origin
$head = git rev-parse HEAD
$remote = git rev-parse origin/main
if ($head -ne $remote) { throw "HEAD is not the pushed remote commit" }

if (git status --porcelain --untracked-files=no) {
  throw "tracked worktree is dirty"
}

git merge-base --is-ancestor e512d90222449b6d6a308be2921ece6465e9219e origin/main
if ($LASTEXITCODE -ne 0) { throw "remote does not contain the review receipt" }

if (Test-Path outputs\p1-a05-size-robustness) {
  throw "formal output already exists"
}

python -m trisched dry-run-p1-a05 `
  --config configs/p1_a05_size_robustness.json
```

最后一条命令只能重新核验无模型 dry-run，不消耗正式训练次数。它必须仍记录两轮 `90/6000` 和三项 `false`。

## 3. 唯一正式训练

预检全部通过后只执行一次：

```powershell
python -m trisched train-p1-a05 `
  --config configs/p1_a05_size_robustness.json
```

正常完成应生成五 seed 的 warm/best/last actor、best/last value、完整 epoch state、curve、validation diagnostics、failure JSONL，以及：

```text
outputs/p1-a05-size-robustness/p1_a05_summary.json
outputs/p1-a05-size-robustness/p1_a05_run_manifest.json
```

如果进程因基础设施原因中断，不得删除目录后重跑，也不得再次使用无 `--resume` 的命令。只有已存在完整 epoch 边界 state 时才允许：

```powershell
python -m trisched train-p1-a05 `
  --config configs/p1_a05_size_robustness.json `
  --resume
```

resume 使用隐藏 staging 完成校验和剩余工作后再交换目录，属于同一正式运行的事务恢复；它不能修改 seed、epoch 或结果选择。

## 4. 唯一正式四切片 Development

训练 summary 的 validation gate、failure 和 illegal action 必须先通过。随后只把 P1-B02 runner 的 masked-MLP checkpoint 根替换为 P1-A05 正式目录，其他 baseline、task-GNN、场景和契约保持不变：

```powershell
python scripts/run_p1_b02_development.py `
  --masked-mlp-dir outputs/p1-a05-size-robustness `
  --output outputs/p1-a05-development-evidence/development-evidence.json

python -m trisched build-report `
  --contract configs/p1_b02_evaluation_contract.json `
  --evidence outputs/p1-a05-development-evidence/development-evidence.json `
  --output outputs/p1-a05-development-report

python scripts/compare_p1_a05_development.py
```

第三条命令不会调度或训练。它严格验证：

- P1-A04 baseline evidence 等于预注册 SHA-256；
- candidate evidence 和标准报告都来自同一干净 commit，且从未访问 public test；
- 五个 candidate checkpoint 的文件/参数 hash 与正式训练 manifest 完全一致；
- baseline/candidate 的四个场景集合、ID 和内容 hash 完全一致；
- 标准报告的零失败与四切片门禁等于从 evidence 独立重算的结果；
- 对每个切片按同一 training seed、同一 scenario 做 `5×30=150` 对精确比较，并使用共享 seed/场景索引做 10,000 次两层配对 bootstrap。

输出为：

```text
outputs/p1-a05-development-comparison/comparison.json
```

已有 comparison 文件时拒绝覆盖，防止把多次候选比较混在一起。

## 5. G3 判定

Development 通过必须同时满足：

| 门禁 | 冻结要求 |
| --- | --- |
| 合法性 | candidate failure/timeout/penalty/illegal 全为 0 |
| 四切片 | candidate 的 ID/size/CCR/system mean ratio 均 `<1.0` |
| size 效应 | size mean `<1.0`，且 candidate−P1-A04 配对 bootstrap CI 上界 `<0` |
| ID guardrail | ID mean `<1.0`，且不高于 P1-A04 ID mean `+0.02` |
| 证据 | checkpoint、summary、manifest、evidence、report 和 comparison hash 全部可重算 |

P1-B02 的 `release_publishable` 在 development 模式按设计恒为 `false`，因为它还要求 final-test receipt。G3 只使用上表 development 门禁；不得为了把 `release_publishable` 变成 `true` 而提前访问 public test。

若任一门禁失败：保留 P1-A04，提交完整负结果，关闭本候选，不再调比例、seed、epoch、学习率或选模规则，不启动 G3，不访问 public test。

若全部通过：状态只能写“允许 B 独立复核”。B 从不可变提交核对五 seed artifact/state、逐 checkpoint validation、正式 evidence/report/comparison，并以独立 review replay 比较去除 wall-clock 后的调度行为。review replay 不得成为第二个候选或替换正式结果；B 签字后才召开 G3。

### B 独立 replay 的固定入口

先确认当前干净 HEAD 与正式 candidate commit 相同，且该提交已包含在本地最新 `origin/main`：

```powershell
git fetch origin
git status --porcelain --untracked-files=no
git merge-base --is-ancestor HEAD origin/main

python scripts/run_p1_b02_development.py `
  --masked-mlp-dir outputs/p1-a05-size-robustness `
  --output outputs/p1-a05-independent-review/development-evidence.json

python -m trisched build-report `
  --contract configs/p1_b02_evaluation_contract.json `
  --evidence outputs/p1-a05-independent-review/development-evidence.json `
  --output outputs/p1-a05-independent-review/development-report

python scripts/compare_p1_a05_development.py `
  --candidate-evidence outputs/p1-a05-independent-review/development-evidence.json `
  --candidate-report outputs/p1-a05-independent-review/development-report/evaluation_report.json `
  --output outputs/p1-a05-independent-review/comparison/comparison.json

python scripts/review_p1_a05_development.py
```

最后一条命令要求正式与 replay comparison 都通过，逐项重算两份 evidence/report manifest 和五 checkpoint，并比较：去除每行 `runtime_ms` 后的完整调度记录、去除 policy runtime/evidence hash 后的报告 JSON、去除 `runtime_*` 列后的三份 CSV、去除 candidate evidence/report 文件 hash 后的配对 comparison。任一非时延字段不同即拒绝，不写 receipt。成功时生成不可覆盖的 `p1_a05_independent_review.json`；G3 授权必须绑定该文件 SHA-256。
