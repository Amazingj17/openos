## 1. 目标与非目标

本任务把公开 STG DAG 接入 TriSched，固定下载字节、转换规则、train/validation/test 名单和逐实例 hash，使后续 teacher、BC/PPO 与 GNN 实验不能通过重新抽样改变 test。原始第三方数据不提交 Git，只提交来源、许可证和冻结清单。

本任务不声称复现 GrapheonRL 完整异构系统，也不导入上游求解器或结果。上游 JSON 含 CPU/GPU feature、核心数和内存约束，而当前 TriSched `Scenario` 不能表达这些约束；因此本次只建立明确标注的 STG topology projection。后续 P1-A01 已通过独立 `train-bc` 路径生成 teacher/checkpoint，原 `pipeline` 仍默认使用合成 smoke。

## 4. TriSched projection v1

转换规则固定为：

- `tasks.Tn.duration` → task `n-1` 的 `workload`；
- `tasks.Tn.dependencies` → 有向边；
- 边的数据量取 source/predecessor task 的 `data`，单位 MB；
- upstream `random_seed` → Scenario seed；
- 上游 `cores`、`memory_required`、`features`、`tags` 和绝对 `system_config` 路径不进入当前 Scenario；
- 资源固定为 device/edge/cloud 三个节点，速度为 1.0/2.4/4.8；
- 链路带宽单位 MB/s：device 对外 100，edge-cloud 10,000，同资源为 `1e9`；固定时延为 0。

该规则使公开 DAG 结构、duration 和 communication volume 可用于当前静态调度器，同时避免伪造当前模型不支持的 GPU/core/memory 约束。任何改变上述规则的提交必须提升 projection version、生成新 manifest，并重新冻结全部实验；禁止在原 v1 名称下静默改动。

## 5. 冻结 split 与防泄漏

rank_key = sha256(utf8("trisched-p1-b01-stg-rnc50-v1" + NUL + source_sha256))

## 6. 复现命令与实际结果

在线首次下载和随后离线复核：

```powershell
python scripts/fetch_stg_benchmark.py
python scripts/fetch_stg_benchmark.py --offline
python -m pytest -o addopts= -q tests/test_benchmark.py
```

在线端到端实际输出：archive 字节数和 SHA-256 一致，180 个源文件/Scenario hash 全部重算通过，split 数量为 120/30/30；离线复跑得到完全相同结果，重建 manifest 与提交文件逐字节一致。聚焦测试为 `5 passed`，最终完整回归为 `163 passed in 4.53s`；覆盖投影语义、非法 dependency、固定 split、防篡改、归档路径穿越和 committed manifest 内部一致性。

B 还对全部 180 个 50-task Scenario 运行 HEFT，并让结果同时通过生产 validator 与独立 validator：180/180 通过；加载/重算 hash 用时约 0.73 秒，调度及双验证约 26.57 秒。时间只表示本机审计成本，不作为算法性能结果。

原合成 smoke 同时回归：pipeline 与 checkpoint evaluate 的 masked MLP `mean_ratio` 均为 1.0，五个 scheduler 的总失败数为 0，checkpoint SHA-256 仍为 `7fb644af6ae06530fdac556fd07fc4281a560646d16c2ab5f00d2641fbd8f2b7`。

## 8. 已知限制

- STG random graphs 是公开合成 benchmark，不是真实生产 trace。
- capability-relaxed projection 不保留 GPU/core/memory 约束，禁止写成完整 GrapheonRL benchmark 复现。
- 原 `pipeline` 尚未切换到该 manifest；P1-A01 已新增只读 train/validation 的 `train-bc` 路径，详见 [HEFT teacher 与 BC 基线](./P1-A01HEFT教师与BC基线.md)。
- 公开 test 名单是“冻结 test”而非竞赛组织方隐藏 test；治理依赖 Git 历史和两人互审。
- 上游 GitHub code 和 Zenodo results 因许可证/rights 不明确而排除；若未来获得明确授权，仍需单独记录版本和许可证后才能接入。
