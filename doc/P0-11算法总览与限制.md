## 统一问题定义

输入是一个静态 DAG、三类异构资源和有向通信代价。每个任务必须选择一个资源和合法开始时间；前驱完成并完成跨资源通信后才可执行，同一资源上的区间不能重叠。当前唯一优化目标是最小化整图 makespan。

所有策略都通过同一 scheduler/evaluator 接口输出完整 schedule，并同时经过生产 validator 与独立 validator。主指标为逐实例 `policy_makespan / HEFT_makespan`；失败、超时和非法调度以冻结惩罚值进入分母，不能删除。

## 算法与用途

| 方法 | 决策 | 主要复杂度 | 当前用途 | 结论 |
| --- | --- | --- | --- | --- |
| HEFT | upward rank 排序，逐任务选最早完成资源 | 约 `O(V²R)`（当前直接实现） | 教师、归一化基线、强启发式 | 正确性已双签 |
| CPOP | 关键路径优先并绑定关键资源 | 约 `O(V²R)` | 独立启发式对照 | 小图轨迹/最优 gap 已核对 |
| Greedy-EFT | 从就绪任务×资源选即时最早完成 | 每步扫描候选 | 弱启发式对照 | 合法但通常劣于 HEFT |
| Random | 在合法 mask 中采样 | 每步扫描合法动作 | 退化/统计 sanity check | 只作下界对照 |
| 精确分支定界 | 穷举 active schedule 并剪枝 | 指数级 | `V≤8` 的 optimum/bound | 不能用于常规训练 |
| legacy MLP | 16 维候选打分，HEFT 模仿 + REINFORCE | 候选数随 `V×R` | 最快 smoke | 复制 HEFT，不是主模型 |
| masked MLP PPO | 14 维候选打分，BC warm-start + PPO/GAE | 每步 `ready×R` 前向与更新 | 当前正式主模型 | ID/CCR 强，size 明显失败 |
| task-GNN PPO | 任务 DAG 双向消息 + 候选打分 | 图消息 + `ready×R` | 单变量表示消融 | 点估计改善但 CI 跨 0，未替换 MLP |
| P1-A05 rollout mixture | 模型不变，只把 PPO rollout 改为等 transition 的 STG-50/synthetic-100 | 仍为两轮、每 seed 12000 transitions | 已关闭的 size 稳健性候选 | 正式 size mean `1.553568>1`，未过门禁，保留 P1-A04 |

## 正式主路为何这样选择

1. HEFT 先提供可手算、可交叉实现的强基线，避免学习模型掩盖环境错误。
2. BC 只负责得到稳定合法 warm-start；PPO 的增量 makespan 奖励严格满足回报和等于终局负 ratio。
3. 正式 PPO 删除直接暴露 HEFT 决策的两个二值特征，防止把教师标签泄漏当成创新。
4. checkpoint selection 只看 validation，候选包含 epoch 0 warm-start；不改善时按预注册规则回退。
5. task-GNN 只改变表示，奖励、split、seed 和训练预算保持不变；配对 CI 跨 0 后停止扩展。
6. size-OOD 失败后只预注册并执行一个等 transition rollout-source 干预；该候选仍未过 size 门禁，已按规则停止，不用重复调参挑选结果。

## 已验证的正确性边界

- 10 个 HEFT 黄金案例、10 个 CPOP/精确 optimum 案例和 40 个固定随机 DAG 已交叉验证；
- 生产与独立 validator 不复用环境的时间计算；遗漏、重复、越界、依赖、通信、资源重叠和 makespan 篡改均会失败；
- PPO 动作始终来自“就绪任务 × 资源”合法 mask；正式证据中非法动作和调度失败均为 0；
- teacher/reference、checkpoint、配置、依赖和产物均有可重算 hash；
- 公开 STG train/validation 与 development ID/OOD 已冻结，公开 test 仍未加载。

## 工程与建模限制

### 环境模型

- 单个 episode 内资源速度、带宽、时延和可用性固定；没有在线任务到达、抢占、迁移和故障恢复。
- 通信按固定矩阵计算；没有链路竞争、拥塞、队列和动态带宽。
- 当前只优化 makespan；没有能耗、费用、SLA、公平性和多目标 Pareto。
- 资源以少量手工属性表达；未建模 GPU 显存、NUMA、容器冷启动或真实云计费。

### 学习方法

- masked MLP 对任务数变化没有显式集合/图归一化，size 泛化证据失败。
- task-GNN 只有一层任务消息传递，资源关系仍靠候选特征；当前统计证据不足。
- PPO 是 on-policy CPU 参考实现，不追求大规模吞吐；断点只支持完整 epoch 边界。
- P1-A05 synthetic-100 与 development size-OOD 的生成器/结构不完全相同，不能解释为纯 size 因果。

### 评测与复现

- public test 有一次性 gate，在 development 全切片通过前禁止访问。
- wall-clock 受机器、Python 和系统负载影响，只在同协议下报告 P50/P95，不参与主性能结论。
- openEuler 证据来自固定用户态容器，宿主内核仍为 WSL2；原生 openEuler VM 尚未验证。
- checkpoint 和正式数据不直接进入源码 Git；最终包必须提供来源、SHA-256 和离线备份。

## 失败时的回退

- 非法调度或失败数大于 0：立即阻断发布，不看均值。
- validation 未改善：选择冻结 warm-start，不追加 epoch。
- 单变量对照 CI 跨 0：保留较简单模型，将新方法标为未证实消融。
- size/development 门禁失败：停止该候选，不访问 public test；新想法必须使用新任务 ID 和新预注册。
- 复现 hash、配置或数据绑定不一致：产物作废并从干净提交重建，禁止手工修补 summary。
