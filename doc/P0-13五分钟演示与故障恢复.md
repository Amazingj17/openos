## 故障恢复路径

### 输入故障

演示固定注入 `tests/fixtures/invalid/scenario_cases.json`。预期：

```json
{"valid": false, "error": {"code": "missing_field", "path": "$.bandwidth", "message": "missing required field 'bandwidth'"}}
```

退出码必须为 2。脚本把它视为预期故障，然后继续 checkpoint-only validation，证明诊断不会污染后续输出。

### 中断训练

正式 PPO 只在完整 epoch 边界恢复 actor/value、两个 Adam、RNG、history 和 best selection；恢复先复制到隐藏 staging，全部成功后才替换正式目录。答辩不现场启动长训练，只展示[P1-03 断点续训证据](./P1-03PPO断点续训设计与验收.md)和对应测试。

### 输出或 hash 损坏

不要手工修 summary/manifest。隔离损坏目录，从绑定 commit、配置和输入重建；checkpoint-only 复评或 run manifest hash 不一致即阻断发布。

### Docker/openEuler 不可用

五分钟主演示不依赖 Docker。平台证据使用预先生成且带 digest 的[P0-10 openEuler 记录](./P0-10openEuler验证记录.md)；Docker 恢复后再执行固定脚本，不能现场换镜像。
