## 发布物边界

当前允许构建的是**源码候选包**，不是最终获奖结果包。源码包包含 Git 跟踪的代码、配置、Schema、测试、文档、CI 和许可证，不包含：

- `outputs/` 下的 checkpoint、训练状态、raw benchmark、archive 和临时证据；
- 公开 STG test 原始字节或一次性授权；
- `.git/`、虚拟环境、缓存、IDE 文件和本机绝对路径；
- `.env`、私钥、证书或 credential 文件。

最终模型包只有在 P1-A05 复核、唯一正式训练、四切片 development、G3 和 P1-06 双签全部通过后才能构建。

## 确定性源码包

构建器直接读取指定 commit 的 Git blobs，不读取未提交工作树。文件按 POSIX 路径排序，zip 时间固定为 1980-01-01，权限固定，内部 `RELEASE_MANIFEST.json` 记录每个源文件的字节数、SHA-256 和 Git blob ID。

```powershell
python scripts/build_release_bundle.py --commit HEAD
python scripts/build_release_bundle.py `
  --verify outputs/release/trisched-source-<commit12>.zip
Get-FileHash outputs/release/trisched-source-<commit12>.zip -Algorithm SHA256
```

同一 commit 连续构建必须得到逐字节相同 zip。构建器遇到 `outputs/`、raw/download 前缀、secret 风险路径、私钥 marker、symlink 或 submodule 时 fail closed；已存在目标文件不会覆盖。

本地主责候选已从干净提交 `9831b805ff96c8b345f02aa7f0651a3236597213` 连续构建两次：两份 zip 均含 129 个 Git 文件，SHA-256 都是 `51e23162cd2dd08d627636ddebf79b7fafc00058d342f65f452496d2c12d7263`，内部 manifest 验证通过。它证明 source bundle 流程可用，不替代 A 的远端独立复核，也不是最终模型包。

## 确定性模型/结果包（工具就绪、执行阻塞）

`scripts/review_p1_a05_development.py` 先把 B 在不可变远端候选提交上的独立 replay 与正式 evidence/report/comparison 做去 wall-clock 精确比较并生成 receipt。随后 `scripts/build_p1_06_bundle.py` 分三步执行：`prepare-review` 从唯一正式训练、Development、comparison、B receipt 和 G3 授权生成不可覆盖的 artifact 清单；A/B 分别重算并签署该清单；`build` 验证两份独立回执后才生成确定性 model/result zip，`verify` 从包内 manifest 重算全部文件。

构建器只接收五个主 actor checkpoint、训练 provenance、P1-A04/P1-A05 Development 证据、标准报告、最小消融 comparison 和治理回执。失败 comparison、缺失/漂移的 B replay、单人/重复回执、checkpoint/report 漂移、脏 tracked 工作树、私钥 marker 或 public-test 路径全部 fail closed。完整 JSON 格式、命令和提交记录规范见[P1-06 最终模型与结果包双签规程](./P1-06最终模型结果包规程.md)。当前不得运行 `prepare-review`，因为正式 P1-A05、独立复核和 G3 尚未发生。

## 回滚与事故处理

- 任一 hash 不一致：隔离整个候选包，从批准 commit 重建，不手工改 manifest。
- source zip 含敏感/禁止文件：撤销发布并轮换可能泄露的 credential；修复规则后使用新版本号。
- CI/openEuler 不一致：阻断 tag，保留失败日志和镜像 digest，不以本机成功覆盖。
- 模型门禁失败：发布功能完整的工程版本和真实负结果，不访问 public test、不宣称性能领先。
