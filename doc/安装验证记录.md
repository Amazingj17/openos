## 2. 许可证决策

选择 Apache License 2.0，原因如下：

- 允许学术、个人和商业使用、修改与再分发；
- 相比只规定版权的宽松许可证，额外包含明确的贡献者专利授权和专利诉讼终止条款；
- 与 Python、PyTorch 及主流开源基础设施的协作方式兼容；
- 便于比赛结束后继续公开开发或与企业合作。

本许可证只覆盖本仓库原创代码和文档。后续引入的论文代码、公开数据、模型和图片必须单独记录来源与许可证，不能因本仓库采用 Apache-2.0 而自动改写第三方许可。

.\.venv\Scripts\python.exe -m pip install --no-build-isolation --no-deps .

## 6. GitHub Actions 设计

CI 在 `main` push、面向 `main` 的 pull request 和手动触发时运行。只修改 `doc/**` 或根目录 Markdown 时不重复运行耗时矩阵。

每个矩阵任务依次执行：

1. 安装锁定依赖；
2. 构建并安装非 editable 项目包；
3. `pip check`；
4. 运行 7 项测试；
5. 运行完整 smoke pipeline；
6. 加载 checkpoint 无训练复评；
7. 检查实例数、合法率和 SHA-256 字段；
8. 上传摘要与逐实例 CSV，保留 14 天。

矩阵：

| 操作系统 | Python 3.10 | Python 3.11 | Python 3.12 |
| --- | --- | --- | --- |
| Ubuntu latest | 必测 | 必测 | 必测 |
| Windows latest | 必测 | 必测 | 必测 |
