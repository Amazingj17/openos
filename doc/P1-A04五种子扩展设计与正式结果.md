## 1. 目标与边界

P1-B02 的机器契约要求主策略使用五个训练 seed。P1-A02 已冻结
`20260717/20260718/20260719` 三个正式 seed，本任务只新增
`20260720/20260721`，不得重新训练或挑选旧 seed。

本任务保持以下合同不变：

- 120 个 train、30 个 validation，公开 test 不加载；
- 14 维 teacher-free MaskedMLPPolicy；
- 相同 BC warm start、PPO/GAE、训练预算、失败惩罚和 validation 选模；
- 不运行正式 development/OOD，不生成 test 性能结果；
- 旧三 seed 的模型、曲线和诊断必须逐字节继承。

旧正式包没有 `seed_*_ppo_training_state.npz`，不能用既有
`--resume` 从三 seed 配置迁移到五 seed。全量重训五个 seed 又会改变已冻结
证据。因此新增显式的 `seed_extension` 模式，而不放宽普通 resume 合同。

## 2. 扩展合同

正式配置为 [`configs/stg_ppo_5seed.json`](../configs/stg_ppo_5seed.json)：

```json
{
  "seeds": [20260717, 20260718, 20260719, 20260720, 20260721],
  "seed_extension": {
    "source_dir": "../outputs/p1-a02-stg-ppo",
    "run_manifest_sha256": "b4320d1bd880434cb16a9bd6cbf69ae441e8ca9fcbdfb8aaefd77de43513f594",
    "reuse_seeds": [20260717, 20260718, 20260719]
  }
}
```

`reuse_seeds` 必须是目标 seed 的非空、严格前缀，且目标至少增加一个
seed。流水线在创建目标目录之前执行以下 preflight：

1. 验证来源目录、run manifest 固定 SHA-256、模式和版本；
2. 逐一重算来源 manifest 声明的全部 artifact 字节数和 SHA-256；
3. 比较来源 resolved config 与目标的训练合同，只允许 seeds、output_dir
   和 seed_extension 不同；
4. 要求来源 config/manifest/summary 的 seed 顺序一致，且
   `test_accessed=false`；
5. 要求每个继承 seed 的 BC、best/last actor/value、曲线、validation
   诊断和失败文件齐全；新格式若有 training state 也一并继承。

目标目录保存来源 manifest 的原字节快照。旧 seed 结果增加
`inherited_verified` provenance，新 seed 增加 `trained_current_run`
provenance；summary 与 run manifest 同时公开继承和训练 seed。普通非扩展
PPO 的输出 schema 不增加空字段。
